## File size limit
**500줄 초과 금지.** 파일이 500줄을 넘으면 컴포넌트·함수·훅으로 분리한다. 예외 없음.

## 작업 기록 워크플로우 (메모리 의존 금지)
기억에만 의존하면 작업내역이 샌다. 일감과 작업내역을 분리해 외부에 남긴다.
- **일감 → GitHub Issues / Projects**: "할 일". 백로그·상태·우선순위는 이슈로 트래킹. 새 작업은 이슈로 먼저 등록.
- **작업내역 → GitHub Wiki** (`작업내역` 페이지): "한 일". 의미 있는 변경을 끝낸 세션 종료 시 날짜·요약·관련 이슈(#번호)를 추가한다. 위키 페이지 원본 템플릿은 repo `wiki/` 폴더 참고.
- 위키는 별도 repo(`<repo>.wiki.git`)라 push 환경이 필요. 직접 접근이 안 되면 `wiki/작업내역.md`에 항목을 추가한 뒤 사용자에게 위키 반영을 안내한다.
- 과거 `DEVLOG.md`(대용량)는 히스토리 보존용으로 그대로 두고, 신규 내역은 위키에 남긴다.

## 타임존 원칙 (위반 시 데이터 오염)
- `client_history.occurred_at` → **순수 UTC** 저장. `to_timestamp(parent_ts::float)` 그대로 사용. `+ INTERVAL '9 hours'` 절대 금지.
- KST 변환은 API/쿼리 레이어에서만: `AT TIME ZONE 'Asia/Seoul'` 또는 날짜 필터에 `+09:00` suffix.
- JS에서 날짜 슬라이스 시 UTC 기준이므로 `toKSTDate()` 변환 후 슬라이스.
- 위반 사례: 과거 에이전트가 +9h를 저장값에 직접 삽입해 166건 오염 발생.
