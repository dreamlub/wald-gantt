# wald-gantt 위키

개인 프로젝트 작업 관리 공간.

- **일감(할 일)** → [GitHub Issues](https://github.com/dreamlub/wald-gantt/issues) · Project 보드
- **작업내역(한 일)** → [작업내역](./작업내역) 페이지
- **과거 기록** → repo `DEVLOG.md` (히스토리 보존, 신규 작성 안 함)

## 기록 규칙
1. 새 작업은 먼저 Issue로 등록 → Project 보드에서 상태 관리.
2. 의미 있는 변경을 끝내면 세션 종료 시 `작업내역`에 날짜·요약·관련 이슈(#번호) 추가.
3. 커밋 메시지는 계속 충실히, "왜/맥락"은 작업내역에 정리.
